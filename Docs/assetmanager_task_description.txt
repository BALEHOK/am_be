AssetManager - приложение, предназначенное для учета ценностей внутри предприятия. 
В качестве основного технического аспекта приложения лежит принцип динамического создания таблиц для хранения экземпляров сущностей.
Таблица DynEntityConfig хранит описания типов, таблица DynEntityAttribConfig - их атрибутов (отношение один ко многим). По данным этих двух таблиц генерируется T-SQL запрос CREATE TABLE.
Имена динамически созданных таблиц начинаются с ADynEntity*.

Поддержка ревизий всех сущностей обеспечивается DynEntityUid, DynEntityId, Revision и ActiveVersion. 
DynEntityUid - уникальный ключ записи. 
DynEntityId - постоянный идентификатор сущности.
Revision - номер ревизии.
ActiveVersion является ли данная ревизия активной версией.

Таким образом, в таблице может быть несколько записей с одним DynEntityId, но разными DynEntityUid и Revision, но при этом только одна из них имеет флаг ActiveVersion.

Основные объекты предметной области - это Asset Type и Asset. Первый - это описание сущности (DynEntityConfig), её свойств (DynEntityAttribConfig). Второй - конкретная сущность (запись в ADynEntity*).
К примеру, тип Car в системе описан следующим набором атрибутов: Model, Chassisnumber, Enginevolume, COemissions и так далее (screenshot-1). 
Сведения о конкретных автомобилях предприятия хранятся в таблице ADynEntityCarZ.

В качестве тестового задания предлагается разработать веб-сайт, отображающий пользовательские типы (Asset Types), а также список сущностей каждого типа. 
Дать возможность просмотреть запись отдельной сущности. Редактирование, удаление не требуется.

Рекомендуется использовать фреймворк ASP.NET MVC. Обязательным является наличие юнит-тестов.
Большой плюс, если в ходе решения задания будут предложены улучшения, касающиеся способа хранения сущностей, индексации базы данных.
